> **这份文档是设计的单一权威。** 任何 UI 决策以此为准。

## 1. 定调(气质)

> 为"个人的、有重量的文学创作与收藏空间"服务。像 **Craft / Bear** 那样克制、精工、每个操作都有质感反馈;像 **Notion / 苹果备忘录** 那样中性、留白、近乎无装饰。**单色为骨,纸墨为肤,留白为呼吸。**

核心是 **减法美学**:能不加就不加。质感来自克制和细节,不来自炫技。

参照系分工:
- **Craft / Bear** —— 质感与重量,精工的卡片/排版
- **Notion** —— 中性的工具感、块结构、大留白
- **苹果备忘录** —— 轻、纸感、系统原生般的无装饰

**减法美学落地为 5 条可执行原则(贯穿所有组件 / 页面,新东西照此判断,不照旧组件抄尺寸):**

1. **纤细而克制的密度** —— 控件小(高 28)、字小(14 / 12)、行紧、线 1px;信息密度高而不挤。质感来自**精确的小尺寸 + 留白**,不来自大块和效果。
2. **不打断** —— 浮层从触发点**就近长出**,不压暗全局、不阻断心流;**摒弃全局 modal**,只有真危险操作(删除)才阻断。
3. **文字优先** —— 界面以**文字**为主,图标小而辅(20px 灰线),少色块、少装饰;颜色只在认知关口出现(见 §3 颜色纲领)。
4. **轻** —— 扁平、柔阴影、无重装饰;不靠效果撑场面。
5. **自适应:布局流动,控件精确** —— 布局(栏宽 / 列数 / 整体)用 `clamp/min/max` + flex/grid **流式自适应**(已在用 `--layout-*` token);**控件尺寸固定 px**(保精致,不随视口缩);**断点换形态**——展示端移动端单栏 + 触摸目标放大(≥40px),管理端桌面优先。

## 2. 隐喻锚点(取舍时"问自己")

- **Liminal / 阈限** —— 内容处在"草稿→定稿→发布"的过渡态;界面要有"之间"的安静感。**这也决定了点缀色的落点(见 §3.3)。**
- **纸墨 / 斑驳** —— 质感来自不完美的颗粒(全站噪点纸纹),不来自光泽。
- **云雾 → 清晰** —— 动效统一是"从模糊/下方凝聚成形"(`enter-rise`),不是花式弹跳。

## 3. 色彩体系(核心)

### 3.0 总原则:克制只施于"装饰",不施于"功能信号"

**先问"用的人那一刻想知道什么",再决定信号;颜色优先作用在文字上,色块能省则省。** 颜色不是拿来"归类"的,是在人需要的认知关口帮他一下。

两条铁律:
1. **文字优先**——能用文字(变色 / 字重)说清的,就不加色块 / 填充。
2. **克制只施于装饰,不施于功能**——装饰(草木)要压;功能信号(危险/成功/选中/可点)要让人一眼可见,该醒目就醒目。把"已发布""未保存"压灰 = 失职。

**三套信号,各一个含义,不串用:**

| 信号 | 人读到的含义 | 用在 |
|---|---|---|
| **accent 色** | 激活 / 主要 / 当前生效 / 进行中 | 主操作(唯一实心块)· 链接·可点(文字)· 选"值"的选中(文字)· focus 焦点 · **进行中/加载(脉动点)** · 开启的开关 |
| **淡灰背景块**(墨系) | hover / 落点 / 我在哪个区域 | hover(菜单/行/按钮)· 选"区域"的选中(侧边栏/Tab/列表当前项:hover 浅、当前深) |
| **语义色文字**(红/绿/橙) | 危险 / 成功 / 警告 | 危险=红字(hover 才淡红底)· 成功=小绿点+文字 · 错误=红字就近;**优先上文字,不糊块** |
| 纯墨文字 | 其余一切 | 读 / 看 / 浏览 |

**选中态准则(最常踩):问"选区域还是选值"** —— 选区域(导航/Tab/列表"我在哪")→ **背景块**;选值(字体/排序"哪个值生效")→ **accent 文字**。

**accent = 第四个功能角色(交互强调)**,含义只有一个:"激活 / 主要"。色值 = **长春花紫 / Very Peri**(Pantone 2022 年度色):daylight `#6667AB`、midnight 提亮 `#9091CE`(hover 各 `#5A5B9C` / `#9D9ED6`)。偏亮优雅、避开所有语义色、呼应纸花圃的薰衣草。**用法以"激活/主要 + 文字优先"为准,不糊色块。**

色彩分层定义如下(所有值 daylight / midnight 双主题,落地写进 `src/index.css` 再经 `@theme` 桥接):

| 墨 | `--ink` | `#1D1D1F` | `#F5F5F7` |

### 3.2 层 1 · 语义色(功能 · 只留 红 / 绿)

状态反馈,只在你**管理 / 编辑**时出现,**优先上文字**(红字、绿点),不糊色块。**只留两个**——警告橙、信息蓝砍掉(草稿 / 未保存 = 灰字、进行中 = **主题色脉动点**(进行中也是"激活",归 accent),都不必专门的语义色)。

| 角色 | 变量 | daylight | midnight | 用途 | 怎么用 |
|---|---|---|---|---|---|
| 危险 | `--danger` | `#D24B3E` | `#F0685A` | 删除确认、保存失败、未保存 | 红**字**;hover 才淡红底;删除确认主按钮可实心红 |
| 成功 | `--success` | `#3F9D57` | `#4FBE6E` | 已发布、已保存、已连接 | 小绿**点** + 文字 |

soft 背景(需要时):`color-mix(in srgb, var(--danger) 12%, transparent)`。

> 砍掉:`--warning` 橙、`--info` 蓝。落地清历史遗留:`--mark-orange`(硬编码 `#f59e0b`)删除;散落的 `rgba(...)` 状态色 → 用 danger/success 的 soft。

### 3.3 层 2 · 草木(实体纸艺元素,不是抽象色)

草木给纯墨界面"添生气",但**它的形态是实体纸艺元素,不是抽象 UI 配色**。

> 已试过并否决的抽象形式:**氛围光晕、标题色条、荧光笔、模块身份色**——抽象的草木色一进 UI 就显脏、显廉价,丢了纸花圃的手工质感。教训:**精致 + 克制 = 实体纸艺,不是涂色。**

**核心:草木 = 状态的隐喻(用"生长"讲状态),不是装饰。** 三态对照 + 高亮:

1. **首页纸花圃 = 已长成** — 实体纸艺花草定格生长动画、百花齐放;**装饰性、不随内容**(空知识库首页也照常开花),首页 UI 本身纯墨。
2. **空状态 = 待生长** — 一方纸艺**空花圃**(土床 + 几株刚破土小苗),小、居中、留白包围。空 = 待播种待生长,与首页"已长成"对照,呼应 Liminal"将生未生"。**全站统一一个素材** `public/garden/empty-garden.webp`,替换各页 lucide 线性图标(`BookOpen` 等)。组件 `shared/EmptyState`。
3. **思考态 = 生长中** — 鸢尾纸艺"**种子→花苞→半开→盛放→淡出→再生**"循环(对应 §7 motion 的"生长";衔接处淡出淡入,不倒放)。素材 `iris-{seed,bud,half,bloom}.webp`;各帧需**显式定大小**保证视觉由小到大单调(否则花苞天生高窄会显最大)。
4. **用户主动高亮(文字记号)** — 编辑器里给文字标记,一组内置呼应的高亮色(草木色 + 长春花紫调),由用户触发,是唯一允许彩色进正文的地方。

   **色板(2026-06-02 定稿)**:4 色 + 「移除」,每色对应文章里的修辞角色,作者按角色选色、读者按色识别角色。色值用 CSS var,暗夜模式自动切提亮版。

   | 色 | 修辞角色 | daylight | midnight | mark value |
   |---|---|---|---|---|
   | 朱批红 | **强调** — 关键字 / 中心句 | `#B85A4E` | `#D67A6E` | `var(--writing-red)` |
   | 长春花紫 | **题旨** — 主题词 / 反复出现的核心意象 | `#6667AB` | `#9091CE` | `var(--writing-purple)` |
   | 灰蓝 | **引述** — 引用 / 对话 / 外部话语 | `#6E8AA0` | `#8AA6BC` | `var(--writing-blue)` |
   | 苔绿 | **注解** — 作者插话 / 解释 / 补充 | `#6E8462` | `#8AA078` | `var(--writing-green)` |

   - 文字色用来**点睛字词**,背景色用来**圈出整句段**;背景色透明度 daylight 0.20、midnight 0.22(用 `var(--writing-{token}-bg)`)
   - 浮动工具栏的 popover **每行直接展示「色点 + 色名 · 角色」**,作者打开即看到全部含义,无需记忆
   - 编辑器组件 `components/ui/writing-highlight-button.tsx`,CSS var 定义在 `src/index.css` 主题块内

**选元素先看花语**(让纸艺元素自带语义,别为做而做):勿忘我 = 记忆(知识库)、鸢尾 = 信使/智慧/希望(写作);颜色尽量呼应主题色长春花紫一脉。

**操作提示按空间分两种**:
- **小行内提示**("加载失败 / 暂无结果"等,空间局促)→ **纯文字,不塞植物**(塞了语义不符,也挤)。
- **大留白区的引导态**(主从布局右侧未选,如"选择一篇笔记开始阅读"——大片空白光一行字太空旷)→ 可配一张**"邀请"语义的纸艺**填充(`garden/reading-invite.webp`,稍繁盛的小花圃,生机舒展、欢迎进入;尺寸克制,与其他空态图相当)。

纸艺空态分三类、语义别混:**待生长空**(有意义但还没内容,如"暂无文集")用空花圃 `empty-garden`;**邀请态**(等你选/进入的大留白)用 `reading-invite`;**纯状态提示**(失败/无结果)纯文字。

**工艺纪律**:简洁手绘纸艺(扁平柔和 + 轻纸纹 + 白底)→ **抠成透明 + 边缘腐蚀去白边 + 无损 webp 输出**(三者缺一暗夜模式都会露白边)。脚本 `scripts/cutout-bg.py`(角点 floodfill 抠白 + `--erode` 去毛刺 + `--trim` 裁白边;输出 webp 时已强制 `lossless+exact`)。

> **暗夜白边根因**:**有损 webp + alpha**。有损 webp 的 RGB 通道独立于 alpha 压缩,锐利 alpha 边缘产生浅色振铃,浏览器 GPU 缩放合成到暗底时露白。坑:用 PIL 单像素/Python 合成黑底都查不出(边缘是干净绿/棕),**只有真·暗夜浏览器渲染才显形**。所有带透明的纸艺素材一律无损 webp;存量改动后必须切到暗夜在浏览器里核验,不能只看文件。

|---|---|---|

| 陶橘 | `#BC8458` | `#D69E6E` |
| 雾紫 | `#87789C` | `#A290B8` |
| 赭黄 | `#B89A4E` | `#CFB46A` |

> 纪律:**UI 系统层面不用抽象草木色**(无光晕 / 无色条 / 无荧光笔 / 无模块身份色)。草木要么是**实体纸艺**(首页一片、空状态统一一株),要么是**用户主动选的高亮**——其余一律纯墨。

## 4. 字体(字族)

> **2026-05-24 定。** 此前只定了字号大小(见 §5 / CLAUDE.md),从未定过字族——全站一直吃系统默认 sans,这是质感空白。本节补齐,以箴言「纸墨」为准。

### 4.1 四类角色,分工明确

| 角色 | 用在 | 字体 | 变量 |
|---|---|---|---|
| **阅读体** | 长文正文:笔记 / 文章 / 文集正文、**Lux 回答**、箴言 | **霞鹜文楷**(`LXGW WenKai`,半正式楷,纸墨手写感) | `--font-reading` |
| **界面体** | UI 控件:导航 / 按钮 / 标签 / 菜单 / 表单 / 元信息 | 系统无衬线(`-apple-system, PingFang SC…`,**不变**) | 默认 |
| **品牌体** | Logo | Alex Brush(手写花体,已有) | — |
| **等宽体** | 代码 | 系统 / 通用 mono | — |

> 为什么阅读体 ≠ 界面体:界面要中性、即时、零加载成本(§1 工具感 / 减法);阅读要纸墨、有重量、有手写温度(箴言)。两者诉求相反,故分治。**界面体永不上 webfont。**

### 4.2 西文:用阅读体字族的原生西文,不另配西文字体

实测把霞鹜文楷的中文配 Newsreader 等"专业"西文,**反而割裂**——楷的笔意与西方衬线的笔形逻辑冲突;霞鹜文楷自带的西文 / 数字是按楷意设计的,**同源协调 > 专业拼凑**。故阅读体的西文 / 数字一律走该字族原生,不配对(`--font-reading` 里只放中文字族 + 系统 serif 兜底,不插西文字体)。

### 4.3 加载策略(务实:中文 webfont 很重)

- **子集化 + 按需**:用 ZeoSeven 自动子集源——霞鹜文楷 = `https://fontsapi.zeoseven.com/292/main/result.css`(family `LXGW WenKai`),浏览器只下用到的字,避免整包 10MB+。
- **`font-display: swap`**:先用系统字垫底,字到了再换,不挡首屏。
- **只给阅读区上字**(经 `--font-reading`),UI 不碰——把体积代价隔离在阅读场景。
- 备选源:jsDelivr `lxgw-wenkai-webfont`(整包,字体样板间 `/design/fonts` 用过;生产不用)。

## 5. 其他维度(沿用并强化 `CLAUDE.md`)

以下规则在 `CLAUDE.md` 已有定义,此处不重复,仅强调它们与本宪法同等效力,逐页对齐时严格执行:

- **字号** —— 只用 Tailwind `text-*` 语义 class,**禁止 inline `fontSize`**;相同语义角色用相同 token。
- **颜色** —— 只用本宪法的 token,**禁止硬编码 hex**(index.css 定义 token 处除外)。
- **间距 / 圆角** —— 分级 token(`rounded-sm/md/lg/xl/2xl` = 6/8/10/12/16px),禁魔法数字。`2xl=16` 为柔和大圆角档(Claude 式 agent 输入框等),2026-05-24 新增。
- **双端一致** —— 展示端 / 管理端相同语义组件,7 项规格(字号/间距/icon/色/形/态/结构)完全一致。
- **组件规模** —— 页面组件 > 300 行必须拆(逻辑抽 hook,展示拆子组件)。

## 7. Agent(Lux)· 箴言落形

> **2026-05-24 定。** Lux 不是去"贴"一套设计——它本就是箴言的时间展开:
> **斑驳而卑微的期许(你抛出的半成形问题)→ 云雾升腾(思考 / 调工具,尚未凝结)→ 斩钉截铁的现实(回答)。**
> 一切 agent 呈现与动效以此为纲;凡新动效种类先写进本节(防漂移铁律)。

### 7.1 呈现(静)

- **回答**:用阅读体(霞鹜文楷,`--font-reading`)+ 纸墨 prose;文字优先,不堆装饰。
- **思考中**:箴言「云雾」态——安静、未凝,**accent 脉动**(§3:进行中 = accent 脉动点),**不要**通用 spinner / 转圈。
- **工具执行**:纸墨**批注**气质——纯墨文字 + 小脉动点;工具显示名沿用**英文**(与后端函数名一致,Claude Code 风);**不要**技术卡片、**不要** iOS 系统色(§3.4 红线,历史上 `ToolCallCard` 误用 `#ff3b30/#30d158`,须用 `--danger/--success`)。

### 7.2 动效(动)—— 扩展 `enter-rise`,三式,**禁花式弹跳 / 炫光**

| 式 | 对应箴言 | 用在 | 手法 |
|---|---|---|---|
| **云雾** | 升腾(未凝) | 思考中 | 极缓 accent 明灭 / 雾感,`--duration-slow` 级 |
| **凝聚** | 模糊 → 清晰 | 回答成形 | 段落 / 消息 `blur→clear` + 微升(复用 `enter-rise` 基因) |
| **生长** | 生机勃勃 | 工具进行 | 安静脉动点 |

> 时长 / 缓动一律 `--duration-*` / `--ease-*` token,禁硬编码魔法值。
