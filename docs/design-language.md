# Liminal Field · 设计语言宪法

> **这份文档是设计的单一权威。** 任何 UI 决策以此为准。
>
> **防漂移铁律:任何"新视觉手法"(新色 / 新动效种类 / 新阴影层级 / 毛玻璃 等)在进入代码前,必须先写进这份文档——先改宪法,再改代码。** 灰色地带的"顺手加一下"就是设计崩坏的起点(参见本项目曾经的 liquid glass 漂移)。

## 1. 定调(气质)

> 为"个人的、有重量的文学创作与收藏空间"服务。像 **Craft / Bear** 那样克制、精工、每个操作都有质感反馈;像 **Notion / 苹果备忘录** 那样中性、留白、近乎无装饰。**单色为骨,纸墨为肤,留白为呼吸。**

核心是 **减法美学**:能不加就不加。质感来自克制和细节,不来自炫技。

参照系分工:
- **Craft / Bear** —— 质感与重量,精工的卡片/排版
- **Notion** —— 中性的工具感、块结构、大留白
- **苹果备忘录** —— 轻、纸感、系统原生般的无装饰

**减法美学落地为 5 条可执行原则(贯穿所有组件 / 页面,新东西照此判断,不照旧组件抄尺寸):**

1. **纤细而克制的密度** —— 控件小(高 28)、字小(14 / 12)、行紧、线 1px;信息密度高而不挤。质感来自**精确的小尺寸 + 留白**,不来自大块和效果。
2. **不打断** —— 浮层从触发点**就近长出**,不压暗全局、不阻断心流;**摒弃全局 modal**,只有真危险操作(删除)才阻断。
3. **文字优先** —— 界面以**文字**为主,图标小而辅(20px 灰线),少色块、少装饰;颜色只在认知关口出现(见 §3 颜色纲领)。
4. **轻** —— 扁平、柔阴影、无重装饰;不靠效果撑场面。
5. **自适应:布局流动,控件精确** —— 布局(栏宽 / 列数 / 整体)用 `clamp/min/max` + flex/grid **流式自适应**(已在用 `--layout-*` token);**控件尺寸固定 px**(保精致,不随视口缩);**断点换形态**——展示端移动端单栏 + 触摸目标放大(≥40px),管理端桌面优先。

## 2. 隐喻锚点(取舍时"问自己")

- **Liminal / 阈限** —— 内容处在"草稿→定稿→发布"的过渡态;界面要有"之间"的安静感。**这也决定了点缀色的落点(见 §3.3)。**
- **纸墨 / 斑驳** —— 质感来自不完美的颗粒(全站噪点纸纹),不来自光泽。
- **云雾 → 清晰** —— 动效统一是"从模糊/下方凝聚成形"(`enter-rise`),不是花式弹跳。

## 3. 色彩体系(核心)

### 3.0 总原则:克制只施于"装饰",不施于"功能信号"

**先问"用的人那一刻想知道什么",再决定信号;颜色优先作用在文字上,色块能省则省。** 颜色不是拿来"归类"的,是在人需要的认知关口帮他一下。

两条铁律:
1. **文字优先**——能用文字(变色 / 字重)说清的,就不加色块 / 填充。
2. **克制只施于装饰,不施于功能**——装饰(草木)要压;功能信号(危险/成功/选中/可点)要让人一眼可见,该醒目就醒目。把"已发布""未保存"压灰 = 失职。

**三套信号,各一个含义,不串用:**

| 信号 | 人读到的含义 | 用在 |
|---|---|---|
| **accent 色** | 激活 / 主要 / 当前生效 / 进行中 | 主操作(唯一实心块)· 链接·可点(文字)· 选"值"的选中(文字)· focus 焦点 · **进行中/加载(脉动点)** · 开启的开关 |
| **淡灰背景块**(墨系) | hover / 落点 / 我在哪个区域 | hover(菜单/行/按钮)· 选"区域"的选中(侧边栏/Tab/列表当前项:hover 浅、当前深) |
| **语义色文字**(红/绿/橙) | 危险 / 成功 / 警告 | 危险=红字(hover 才淡红底)· 成功=小绿点+文字 · 错误=红字就近;**优先上文字,不糊块** |
| 纯墨文字 | 其余一切 | 读 / 看 / 浏览 |

**选中态准则(最常踩):问"选区域还是选值"** —— 选区域(导航/Tab/列表"我在哪")→ **背景块**;选值(字体/排序"哪个值生效")→ **accent 文字**。

**accent = 第四个功能角色(交互强调)**,含义只有一个:"激活 / 主要"。色值 = **长春花紫 / Very Peri**(Pantone 2022 年度色):daylight `#6667AB`、midnight 提亮 `#9091CE`(hover 各 `#5A5B9C` / `#9D9ED6`)。偏亮优雅、避开所有语义色、呼应纸花圃的薰衣草。**用法以"激活/主要 + 文字优先"为准,不糊色块。**

色彩分层定义如下(所有值 daylight / midnight 双主题,落地写进 `src/index.css` 再经 `@theme` 桥接):

### 3.1 层 0 · 底色(黑白灰纸墨 · 锁死 · ~95% 界面)

界面的"底",绝对主导。**已锁定,不在设计讨论范围内。**

| 角色 | 变量 | daylight | midnight |
|---|---|---|---|
| 纸 | `--paper` | `#FFFFFF` | `#161617` |
| 纸·暗 | `--paper-dark` | `#F5F5F7` | `#1C1C1E` |
| 架 | `--shelf` | `rgba(0,0,0,.04)` | `rgba(255,255,255,.06)` |
| 墨 | `--ink` | `#1D1D1F` | `#F5F5F7` |
| 墨·浅 | `--ink-light` | `#424245` | `#D1D1D6` |
| 墨·淡 | `--ink-faded` | `#6E6E73` | `#98989D` |
| 墨·幽 | `--ink-ghost` | `#AEAEB2` | `#48484A` |
| 分隔 | `--separator` | `rgba(60,60,67,.12)` | `rgba(84,84,88,.36)` |
| 盒边 | `--box-border` | `rgba(0,0,0,.06)` | `rgba(255,255,255,.08)` |

### 3.2 层 1 · 语义色(功能 · 只留 红 / 绿)

状态反馈,只在你**管理 / 编辑**时出现,**优先上文字**(红字、绿点),不糊色块。**只留两个**——警告橙、信息蓝砍掉(草稿 / 未保存 = 灰字、进行中 = **主题色脉动点**(进行中也是"激活",归 accent),都不必专门的语义色)。

| 角色 | 变量 | daylight | midnight | 用途 | 怎么用 |
|---|---|---|---|---|---|
| 危险 | `--danger` | `#D24B3E` | `#F0685A` | 删除确认、保存失败、未保存 | 红**字**;hover 才淡红底;删除确认主按钮可实心红 |
| 成功 | `--success` | `#3F9D57` | `#4FBE6E` | 已发布、已保存、已连接 | 小绿**点** + 文字 |

soft 背景(需要时):`color-mix(in srgb, var(--danger) 12%, transparent)`。

> 砍掉:`--warning` 橙、`--info` 蓝。落地清历史遗留:`--mark-orange`(硬编码 `#f59e0b`)删除;散落的 `rgba(...)` 状态色 → 用 danger/success 的 soft。

### 3.3 层 2 · 草木(实体纸艺元素,不是抽象色)

草木给纯墨界面"添生气",但**它的形态是实体纸艺元素,不是抽象 UI 配色**。

> 已试过并否决的抽象形式:**氛围光晕、标题色条、荧光笔、模块身份色**——抽象的草木色一进 UI 就显脏、显廉价,丢了纸花圃的手工质感。教训:**精致 + 克制 = 实体纸艺,不是涂色。**

草木的三个正确归宿:

1. **首页纸花圃** — 实体纸艺花草(定格生长动画),首页专属、百花齐放(生长"已完成")。**首页 UI 本身保持纯墨**(纸花圃已经够花,不再叠加点缀)。
2. **空页面 / 空状态** — 放**统一的一株**纸花圃同源纸艺植物(实体图),小、居中、留白包围。
   - 精致(纸艺质感)+ 克制(一株,且全站**统一同一个素材**,不是每页换花——多种花就是新的"乱")+ 有隐喻(空 = 待生长,与首页"已长成"对照,呼应 Liminal "将生未生")。
   - 替换现有的 lucide 线性图标(`BookOpen` 等通用 UI 味)。
   - **素材待办**:用 GPT 生成**一个**空状态专属纸艺元素(嫩芽/种子/单株),全站通用;现阶段先用 `public/garden/*.webp` 占位。
3. **用户主动高亮(文字记号)** — 编辑器里给文字标记,提供一组**内置、与整体呼应的高亮色**(纸墨气质柔和色:草木色 + 长春花紫调,像 Notion 文字高亮但更纸墨)。由用户触发,是唯一允许彩色进正文的地方;具体色板待做编辑器高亮时拉出来定。

草木色板(仅供"用户高亮"与纸艺素材的色调参考),晨雾草木·原版:

| 角色 | daylight | midnight |
|---|---|---|
| 苔绿 | `#6E8462` | `#8AA078` |
| 陶橘 | `#BC8458` | `#D69E6E` |
| 雾紫 | `#87789C` | `#A290B8` |
| 赭黄 | `#B89A4E` | `#CFB46A` |
| 灰蓝 | `#6E8AA0` | `#8AA6BC` |

> 纪律:**UI 系统层面不用抽象草木色**(无光晕 / 无色条 / 无荧光笔 / 无模块身份色)。草木要么是**实体纸艺**(首页一片、空状态统一一株),要么是**用户主动选的高亮**——其余一律纯墨。

### 3.4 已废弃 / 禁用(防止回流)

- ❌ **毛玻璃 / backdrop-filter**——全站清除(面板毛玻璃改实色 `--paper`+阴影+边框;弹窗遮罩改纯暗化 `rgba(0,0,0,.4)`,不模糊)。`--glass-*` token 删除。
- ❌ **荧光笔 / 亮档点缀**——评估后不采用。
- ❌ **抽象草木色进 UI**(氛围光晕 / 标题色条 / 模块身份色)——显脏，否决;草木只以实体纸艺或用户高亮存在(见 §3.3)。
- ❌ **iOS 系统色**(`#FF3B30` 等鲜艳系统色)——由 §3.2 醒目档替换。

## 4. 其他维度(沿用并强化 `CLAUDE.md`)

以下规则在 `CLAUDE.md` 已有定义,此处不重复,仅强调它们与本宪法同等效力,逐页对齐时严格执行:

- **字号** —— 只用 Tailwind `text-*` 语义 class,**禁止 inline `fontSize`**;相同语义角色用相同 token。
- **颜色** —— 只用本宪法的 token,**禁止硬编码 hex**(index.css 定义 token 处除外)。
- **间距 / 圆角** —— 分级 token(`rounded-sm/md/lg/xl` = 6/8/10/12px),禁魔法数字。
- **双端一致** —— 展示端 / 管理端相同语义组件,7 项规格(字号/间距/icon/色/形/态/结构)完全一致。
- **组件规模** —— 页面组件 > 300 行必须拆(逻辑抽 hook,展示拆子组件)。
