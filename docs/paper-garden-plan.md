## 三、代码架构

### 组件结构

```tsx
<PaperGarden>
  <SoilLayer />                          ← soil.png
  <GrassField count={35} />              ← 4 种草 × 随机排列
  <PaperFlower species="rose"       x={25%} delay={2.0} />
  <PaperFlower species="lavender"   x={40%} delay={3.0} />
  <PaperFlower species="bell"       x={15%} delay={3.8} />
  <PaperFlower species="daisy"      x={60%} delay={4.5} />
  <PaperFlower species="dandelion"  x={75%} delay={5.2} />
</PaperGarden>
```

### 代码对每个素材做什么

| 操作 | 说明 |
|------|------|
| **位置随机** | x 在土壤范围内随机偏移 |
| **缩放随机** | 0.7x ~ 1.3x，增加自然感 |
| **旋转随机** | -15° ~ +15°，避免排列感 |
| **镜像翻转** | 50% 概率 scaleX: -1 |
| **入场动画** | scale 0→1（从底部锚点），带 blur 雾散 |
| **花头切换** | crossfade: 花苞图 → 全开图 |

### 确定性随机

使用 seeded random（现有 `seededRng` 函数），保证每次刷新布局一致。

## 四、动画时间线

```
时间    事件
─────────────────────────────────────
0.0s    土壤淡入 + 从下方升起（0.5s）
0.5s    草叶开始逐根冒出
        ├── 30-40 根，每根 scale 0→1
        ├── 底部锚点，像从土里钻出来
        ├── 随机延迟间隔 0.02-0.05s
        └── 带 blur 雾散效果
2.0s    第一朵花（玫瑰）开始生长
        ├── 茎 scaleY 0→1（0.5s）
        ├── 叶片 scale 0→1 + rotate（0.3s）
        ├── 花苞出现
        └── crossfade 花苞→半开→全开（0.8s）
3.0s    第二朵花（薰衣草）
3.8s    第三朵花（风铃）
4.5s    第四朵花（雏菊）
5.2s    第五朵花（蒲公英）
7.0s    全部绽放完成
7.5s    最后一朵微微晃动（±2° rotate）
8.0s    静止。完成。
```

## 五、单朵花生长动画

以玫瑰为例，由 4 个 `<img>` 叠加：

```
图层（从上到下）：
  [花头]  ← position: absolute, 锚定在茎顶端
  [叶片右] ← 旋转展开
  [叶片左] ← 旋转展开（镜像）
  [花茎]  ← scaleY 动画，bottom 锚点
```

时序：
```
+0.0s   茎开始生长 scaleY 0→1（0.5s, ease-out）
+0.3s   左叶展开 scale 0→1, rotate -30°→0°（0.3s）
+0.4s   右叶展开 scale 0→1, rotate 30°→0°（0.3s）
+0.5s   花苞淡入（0.2s）
+0.8s   花苞 → 半开 crossfade（0.4s）
+1.2s   半开 → 全开 crossfade（0.4s）
```

单朵花完整生长：约 **1.5s**。

## 六、生图顺序和优先级

```
第一批 ──── 草叶 G1-G4（4 张）
            最先需要，搭框架后马上能看到草地效果

第二批 ──── 花茎 T1-T2 + 叶片 L1-L2（4 张）
            结构件，各花通用

第三批 ──── 花朵 F1-F5（11 张）
            最花精力，每种花 2-3 个阶段
            建议按顺序：玫瑰 → 雏菊 → 薰衣草 → 风铃 → 蒲公英
            每种确认风格后再下一种
```
